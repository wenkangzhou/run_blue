### 首屏一句话卖点

> **你的 Strava 数据，按你的方式重新呈现。**
>
> Strava 订阅才能看的高级分析、AI 训练建议、周期化课表——这里全部免费。

### 核心功能模块（5 个）

#### 模块 1：AI 训练分析（核心卖点，放第一个）
```
🤖 AI 教练 — 深度读懂你的每一次训练

基于 Kimi API（Moonshot），结合 Daniels 配速区间和 Joe Friel 心率区间：
· 配速区间定位（E/M/T/I/R）
· 心率区间分布可视化（基于 LTHR 的 Z1-Z5 彩色条形图）
· 配速模式识别（热身冷身 / 间歇 / 渐进跑 / 比赛跑崩检测）
· 心率漂移分析（区分正常加速段 vs 真正漂移）
· 训练负荷评估（结合单次强度 + 周跑量趋势）
· 同类型历史对比（碾压了多少过去的自己）
· 针对性恢复建议与训练建议
· 结合身高体重 / BMI 的身体构成建议
· 比赛场景专项分析（赛后恢复、深度疲劳预警）

全部基于你的真实数据，不是模板套话。
```

#### 模块 2：数据总览 — 活动列表与搜索
```
📊 你的跑步数据中心

· 搜索框快速定位历史活动
· 周/月/年数据快捷预览（本周跑了多少，一目了然）
· 按周分组的活动网格卡片，路线图预览 + 距离/时间
· 月份标签快速识别时间线
· 深色/浅色主题自动适配

像浏览相册一样浏览你的跑步历史。
```

```
· 大字报统计数字滚动（总里程 / 总次数 / 总时间 / 爬升）
· 全量轨迹暗色地图（全部活动叠加，年份筛选自动适配视角）
· 年度对比柱状图（按年堆叠跑量趋势）
· GitHub 风格活动日历热力图
· 年度归档折叠列表
· 分享给朋友，像展示作品一样展示你的跑步生涯

RUNNER_PROFILE v1.0 // BUILT WITH STRAVA DATA // NO LOGIN REQUIRED
```

#### 模块 4：AI 训练计划生成器
```
📋 输入目标，输出课表

选择 5K / 10K / 半程 / 全程马拉松，AI 自动生成：
· 16-20 周周期化训练计划
· 基础期→建立期→巅峰期→减量期自动切换
· 每周课表含配速、距离、训练类型（轻松跑/间歇/阈值/长距离/力量训练）
· 巅峰期/减量期自动加入马拉松配速（M配速）段落

配速自动基于你的 PB 计算，每一公里都有明确目标。
```

### 开发者说

```
💡 为什么做跑蓝

作为一名跑了 12 年、累计 11000+ 公里的跑者，我对 Strava 的付费订阅越来越不满：
· 想看心率区间分析？订阅。
· 想看训练负荷和恢复建议？订阅。
· 想看周期化训练计划？订阅。
· 想按自己的习惯浏览数据？做不到。

跑蓝的初心很简单：基于 Strava 的跑步数据，按跑者自己的习惯重新呈现。

Strava 擅长记录，但它不擅长"读懂"你的训练。跑蓝用 AI 补上这块：
· 分析你的心率区间、配速模式、训练负荷
· 识别跑崩风险、给出恢复建议
· 生成周期化训练课表

所有这些，不需要 Strava 付费订阅。

跑蓝是一个开源自部署项目。你需要自行注册 Strava 应用和申请 Moonshot API Key，
部署到自己的域名上。详见 GitHub 仓库的自部署指南。

开源协议：MIT
```

## 五、图片加工建议

### 尺寸规格
- **轮播图**：iPhone 竖屏比例，推荐 1170×2532（或按比例缩放）
- **功能模块图**：宽度 800-1200px
- 用浏览器 DevTools 设备模拟截图，再按比例裁切

### 加工规范
1. **文字位置**：放在图片下半部分，避开主要 UI 内容
2. **文字样式**：
   - 标题：24-32px，白色或品牌色
   - 副标题：16-18px，灰色
   - 底部叠加 `backdrop-blur` 毛玻璃效果底栏
3. **不要遮挡关键 UI**：AI 分析的心率条形图、热力图等核心数据要完整露出
4. **统一风格**：深色主题截图优先，与像素极简风格一致
